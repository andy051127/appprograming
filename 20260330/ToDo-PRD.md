# PRD: 개인 할 일 관리 앱

---

## 1. 개요

| 항목 | 내용 |
|------|------|
| 제품명 | Todo Manager |
| 목적 | 개인 일상 업무를 효율적으로 관리 |
| 대상 사용자 | 개인 사용자 (1인) |
| 플랫폼 | 웹 브라우저 (데스크탑 우선) |
| 기술 스택 | HTML, CSS, Vanilla JavaScript, LocalStorage |

---

## 2. 문제 정의

하루 10~20개의 할 일을 관리할 때 다음 문제가 발생한다:
- 카테고리 없이 나열되면 우선순위 파악이 어려움
- 완료한 항목과 미완료 항목이 섞여 진행 상황 파악이 어려움
- 앱을 닫으면 데이터가 사라짐

---

## 3. 목표 (Goals)

- 할 일을 30초 이내에 추가할 수 있다
- 카테고리별로 할 일을 분류해 한눈에 파악한다
- 오늘의 진행률을 즉시 확인한다
- 브라우저를 닫아도 데이터가 유지된다

---

## 4. 기능 요구사항

### 4.1 할 일 CRUD

| 기능 | 설명 | 우선순위 |
|------|------|---------|
| 추가 | 텍스트 입력 후 Enter 또는 버튼 클릭으로 등록 | P0 |
| 수정 | 항목 클릭 시 인라인 편집 | P0 |
| 삭제 | 삭제 버튼으로 개별 삭제 | P0 |
| 완료 체크 | 체크박스 클릭으로 완료/미완료 토글 | P0 |

### 4.2 카테고리

- **업무 / 개인 / 공부** 3가지 고정 카테고리
- 할 일 추가 시 카테고리 선택 (기본값: 업무)
- 카테고리별 필터링 뷰 제공

### 4.3 진행률

- 전체 진행률: `완료 수 / 전체 수 × 100%` 프로그레스 바
- 카테고리별 완료 건수 표시

### 4.4 데이터 영속성

- `localStorage`에 자동 저장
- 페이지 로드 시 자동 복원
- 별도 저장 버튼 없음 (자동)

---

## 5. 비기능 요구사항

| 항목 | 기준 |
|------|------|
| 응답속도 | 모든 UI 인터랙션 100ms 이내 |
| 호환성 | Chrome, Firefox, Edge 최신 버전 |
| 파일 구조 | 단일 `index.html` 파일로 배포 가능 |
| 외부 의존성 | 없음 (CDN 포함 금지) |

```javascript
  }

```

- 할 일 추가 → 완료 흐름을 3번의 클릭/입력 이내로 처리
- 새로고침 후 데이터 100% 유지
