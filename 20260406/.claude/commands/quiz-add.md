quiz-game/data/questions.js 파일을 읽고, 새로운 퀴즈 문제를 생성하여 추가하라.

## 입력값 처리

$1 → 카테고리, $2 → 난이도로 사용한다.

**카테고리 유효값:** `한국사` | `과학` | `지리` | `일반상식`
**난이도 유효값:** `easy` | `medium` | `hard` | `초급` | `중급` | `고급`

- `초급` → `easy`, `중급` → `medium`, `고급` → `hard` 로 변환하여 처리
- $1이 없거나 유효하지 않으면: `"카테고리를 입력해주세요. 사용법: /quiz-add [카테고리] [난이도]"` 출력 후 중단
- $2가 없거나 유효하지 않으면: `"난이도를 입력해주세요. 사용법: /quiz-add [카테고리] [난이도]"` 출력 후 중단

---

## Step 1 — 현황 파악

questions.js를 읽어 아래를 확인하라:

1. 현재 전체 문제 수와 마지막 ID 번호
2. 해당 카테고리의 현재 문제 수와 서브토픽별 분포
3. 해당 카테고리+난이도 조합의 현재 문제 수

현황을 아래 형식으로 출력하라:
```
📋 현황
- 전체 문제 수 : N개  (마지막 ID: N)
- [카테고리] 문제 수 : N개
  서브토픽 분포: 서브토픽1(N) / 서브토픽2(N) / ...
- [카테고리] + [난이도] : 현재 N문제
```

---

## Step 2 — 서브토픽 결정

해당 카테고리의 서브토픽 목록 (questions.js의 기존 subtopic 값 기준):

| 카테고리 | 서브토픽 |
|----------|---------|
| 한국사 | 고대, 고려, 조선, 근현대 |
| 과학 | 물리, 화학, 생물, 지구과학 |
| 지리 | 세계수도, 지형·자연환경, 국가·문화권, 한국지리 |
| 일반상식 | 사회·경제, 문화·예술, 스포츠, 시사·과학기술 |

현재 해당 카테고리에서 문제가 가장 적은 서브토픽을 선택하여 새 문제의 subtopic으로 사용하라.

---

## Step 3 — 문제 생성

새 문제를 1개 생성하라. 반드시 아래 검증 가이드라인을 모두 통과해야 한다.

### 검증 가이드라인 (4개 항목 전부 충족 필수)

**① 단일 정답 원칙**
- 정답이 단 하나여야 한다
- 다른 해석이 가능한 경우 조건을 명시한다 (예: "2024년 기준", "면적 기준")

**② 최상급 표현 기준 명시**
- "가장, 최초, 최대, 최소, 최고, 최장, 최단, 최다, 유일, 처음, 제일" 표현 사용 시
  측정 기준을 문제 텍스트에 반드시 포함한다
- 기준 명시 예: "면적 기준으로", "전통적인 측정 기준으로", "연간 관람객 수 기준"

**③ 시간·범위 명확화**
- 시간이 지나면 바뀔 수 있는 정보는 기준 시점을 명시한다
- 지리적·분류적 범위가 불명확하면 한정한다 (예: "남북한을 포함한 한반도에서")

**④ 사실 정확성**
- 논란이 있는 내용은 주류 학설 기준으로 작성하고 explanation에 명시한다
- 수치(연도, 길이, 면적 등)는 정확한 값을 사용한다

### 생성 형식

아래 스키마를 정확히 따른다:

```js
{
  id: [마지막 ID + 1],
  category: "$1",
  subtopic: "[결정된 서브토픽]",
  difficulty: "[easy|medium|hard]",
  question: "[문제 텍스트]",
  choices: ["선택지1", "선택지2", "선택지3", "선택지4"],
  answer: [0~3 사이 정수, 0-based],
  explanation: "[해설 텍스트]"
}
```

- `answer`는 **셔플 전 원본 choices 배열** 기준 0-based 인덱스여야 한다
- 선택지는 정답이 특정 위치에 편중되지 않도록 고르게 배치한다 (0, 1, 2, 3 중 선택)
- explanation은 정답 이유와 오답이 틀린 이유를 간략히 포함한다

---

## Step 4 — 자체 검증

생성한 문제를 파일에 추가하기 전에 아래 체크리스트를 직접 점검하고 결과를 출력하라:

```
🔍 자체 검증
- [ ] 정답이 단 하나인가?
- [ ] 최상급 표현 사용 시 기준이 문제 텍스트에 명시되어 있는가?
- [ ] 시간·범위가 명확한가?
- [ ] 수치와 사실이 정확한가?
- [ ] answer 인덱스가 choices 배열의 정답 항목과 일치하는가?
- [ ] explanation이 정답 근거를 충분히 설명하는가?
```

체크리스트에서 하나라도 실패하면 문제를 수정한 후 다시 점검하라. 모두 통과한 후에만 Step 5로 진행한다.

---

## Step 5 — 파일 추가

검증을 통과한 문제를 questions.js의 **해당 카테고리 블록 마지막 문제 바로 뒤**에 삽입하라.

삽입 위치 주석 형식:
```js
  // [서브토픽] — [easy|medium|hard]
  { ... 새 문제 객체 ... },
```

---

## Step 6 — 결과 출력

파일 수정 후 아래 형식으로 결과를 출력하라:

```
✅ 문제 추가 완료

ID       : [번호]
카테고리 : [카테고리]
서브토픽 : [서브토픽]
난이도   : [🟢 초급 | 🟡 중급 | 🔴 고급]
정답     : [①②③④] [정답 텍스트]

문제: [문제 텍스트]
선택지:
  ① [선택지1]
  ② [선택지2]
  ③ [선택지3]
  ④ [선택지4]
해설: [explanation]

전체 문제 수: N개 → N+1개
```
