quiz-game/data/questions.js 파일을 읽고 아래 지침에 따라 문제를 검증하라.

## 검증 대상 범위

$ARGUMENTS 값을 확인하라:
- 값이 있으면 (예: `한국사`, `과학`, `지리`, `일반상식`) → 해당 카테고리의 문제만 검증
- 값이 없으면 → 전체 40문제 모두 검증

## 검색 대상 표현

각 문제의 `question` 필드에서 아래 표현을 찾아라:

> 가장, 최초, 최대, 최소, 최고, 최장, 최단, 최다, 최강, 유일, 처음, 제일

## 출력 형식

### 1. 검증 범위 안내

맨 위에 아래 형식으로 출력:
- 카테고리 지정 시: `검증 범위: [카테고리명] (N문제)`
- 전체 검증 시: `검증 범위: 전체 (40문제)`

### 2. 결과 표

| ID | 카테고리 | 난이도 | 문제 텍스트 | 포함된 표현 | 기준 명시 여부 |
|----|----------|--------|------------|------------|---------------|

- **기준 명시 여부**: 문제 텍스트에 측정 기준이 명시되어 있으면 ✅, 없으면 ⚠️
  - ✅ 예시: "최대 수심 기준으로", "면적이", "전통적인 측정 기준으로", "연간 관람객 수 기준"
  - ⚠️ 예시: 단순히 "가장 깊은", "가장 큰" 처럼 기준 없이 최상급만 사용

### 3. 기준 추가 권장 목록

⚠️ 항목이 하나라도 있으면, 표 아래에 항목별로 구체적인 수정 제안을 출력하라:

```
⚠️ ID [번호] — [원래 문제 텍스트]
   포함 표현 : [해당 표현]
   명시 권장 : [어떤 기준을 추가해야 하는지 구체적으로 설명]
   수정 예시 : [수정된 문제 텍스트]
```

### 4. 요약

마지막에 아래 형식으로 출력:
```
검증 완료: 총 N개 문제 중 최상급 표현 M개 발견 (✅ K개 기준 명시 / ⚠️ J개 기준 누락)
```

최상급 표현이 하나도 없으면 `"해당 범위에 최상급 표현이 포함된 문제가 없습니다."`라고 출력하라.
