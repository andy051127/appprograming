quiz-game/data/questions.js 파일을 읽고, ID $1번부터 $2번까지의 문제를 검토하라.

$1 또는 $2가 입력되지 않았으면 다음과 같이 처리하라:
- $1만 없으면: "시작 ID를 입력해주세요. 사용법: /quiz-range [시작ID] [끝ID]" 출력 후 중단
- $2만 없으면: $1 단일 문제만 검토
- 둘 다 없으면: "사용법: /quiz-range [시작ID] [끝ID]  예) /quiz-range 1 10" 출력 후 중단
- $1이 $2보다 크면: "시작 ID는 끝 ID보다 작거나 같아야 합니다." 출력 후 중단
- 범위 내에 존재하지 않는 ID가 있으면 건너뛰고 존재하는 문제만 검토

---

## 1. 검토 범위 안내

맨 위에 출력:
```
검토 범위: ID $1 ~ $2  |  대상 문제 수: N개  |  카테고리: [포함된 카테고리 목록]
```

---

## 2. 문제 목록

각 문제를 아래 형식으로 출력하라:

| ID | 카테고리 | 서브토픽 | 난이도 | 정답 번호 | 문제 텍스트 |
|----|----------|----------|--------|-----------|------------|

- 난이도: easy → 🟢 초급 / medium → 🟡 중급 / hard → 🔴 고급
- 정답 번호: answer 값(0-based)을 1-based로 변환하여 `①②③④` 기호로 표시

---

## 3. 난이도 분포

```
🟢 초급 (easy)  : N문제  [██████░░░░]  X%
🟡 중급 (medium): N문제  [██████░░░░]  X%
🔴 고급 (hard)  : N문제  [██████░░░░]  X%
```

- 막대 길이는 10칸 기준, 비율에 따라 █ 채움
- 권장 비율은 초급 40% / 중급 40% / 고급 20% 이며, 벗어나면 ⚠️ 경고 표시

---

## 4. 정답 분포

```
① 번 정답: N문제  [██████░░░░]  X%
② 번 정답: N문제  [██████░░░░]  X%
③ 번 정답: N문제  [██████░░░░]  X%
④ 번 정답: N문제  [██████░░░░]  X%
```

- 특정 번호에 정답이 50% 이상 몰려 있으면 ⚠️ 경고 표시
- 정답이 0개인 번호가 있으면 ⚠️ 경고 표시

---

## 5. 카테고리별 요약

범위 내 문제가 2개 이상의 카테고리에 걸쳐 있을 때만 출력:

| 카테고리 | 문제 수 | 초급 | 중급 | 고급 |
|----------|---------|------|------|------|

---

## 6. 종합 진단

아래 항목을 순서대로 점검하고 결과를 출력하라:

- [ ] 난이도 분포가 권장 비율(초급 40% / 중급 40% / 고급 20%)에 근접한가?
- [ ] 정답이 특정 번호에 편중되지 않았는가? (한 번호 50% 이상이면 편중)
- [ ] 정답이 0개인 선택지 번호가 없는가?

이상이 없으면: `✅ 이상 없음`
이상이 있으면: 각 항목을 `⚠️ [내용]` 형식으로 나열하고 개선 방향을 제안하라.
