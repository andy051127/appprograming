# 상식 퀴즈 게임 — 단계별 구현 확인 체크리스트

---

## Step 1 — 문제 데이터 & 프로젝트 기반 구조

### 프로젝트 구조
- [ ] 1. `quiz-game/` 루트 폴더가 생성되어 있다
- [ ] 2. `index.html`, `css/style.css`, `data/questions.js`, `js/game.js`, `js/quiz.js`, `js/leaderboard.js` 6개 파일이 모두 존재한다
- [ ] 3. `index.html`에서 `style.css`와 4개의 js 파일이 올바른 순서로 로드된다 (`questions.js → game.js → quiz.js → leaderboard.js`)

### HTML 화면 구조
- [ ] 4. `#screen-home`, `#screen-quiz`, `#overlay-feedback`, `#screen-result`, `#screen-leaderboard` 5개의 div가 존재한다
- [ ] 5. 초기 로드 시 `#screen-home`만 표시되고 나머지 4개는 `.hidden` 상태다
- [ ] 6. 피드백 오버레이(`#overlay-feedback`)가 문제 화면(`#screen-quiz`) 위에 겹치도록 마크업 구조가 설계되어 있다

### CSS 기반 스타일
- [ ] 7. CSS 변수 5개(`--color-primary`, `--color-correct`, `--color-wrong`, `--color-bg`, `--color-card`)가 `:root`에 정의되어 있다
- [ ] 8. `box-sizing: border-box` reset과 기본 margin/padding 초기화가 적용되어 있다
- [ ] 9. `.hidden { display: none }` 클래스가 정의되어 있다
- [ ] 10. `.card` 공통 컴포넌트와 `.btn`, `.btn-primary`, `.btn-secondary` 버튼 스타일이 정의되어 있다
- [ ] 11. 모바일 기본 레이아웃이 `max-width: 480px`, 중앙 정렬로 설정되어 있다

### 문제 데이터
- [ ] 12. `data/questions.js`에 총 40개의 문제 객체가 있다
- [ ] 13. 각 문제 객체에 `id`, `category`, `subtopic`, `difficulty`, `question`, `choices`, `answer`, `explanation` 8개 필드가 모두 존재한다
- [ ] 14. 카테고리별 정확히 10문제씩(한국사 10, 과학 10, 지리 10, 일반상식 10) 존재한다
- [ ] 15. `answer` 값이 모든 문제에서 `0~3` 범위의 정수다
- [ ] 16. `difficulty` 값이 모든 문제에서 `"easy"`, `"medium"`, `"hard"` 중 하나다
- [ ] 17. 한국사 서브토픽 배분이 고대 2, 고려 2, 조선 3, 근현대 3으로 맞다
- [ ] 18. 과학 서브토픽 배분이 물리 3, 화학 2, 생물 3, 지구과학 2로 맞다
- [ ] 19. 지리 서브토픽 배분이 세계수도 3, 지형·자연환경 3, 국가·문화권 2, 한국지리 2로 맞다
- [ ] 20. 일반상식 서브토픽 배분이 사회·경제 3, 문화·예술 2, 스포츠 2, 시사·과학기술 3으로 맞다

## Step 2 — 게임 핵심 로직 & 상태 관리

### 상태 관리 (game.js)
- [ ] 1. `STATE` 객체에 `IDLE`, `PLAYING`, `FEEDBACK`, `RESULT`, `LEADERBOARD` 5개 상태가 정의되어 있다
- [ ] 2. `startGame(category, difficulty)` 호출 시 `GameSession`이 초기화되고 상태가 `PLAYING`으로 전환된다
- [ ] 3. `difficulty: "mixed"` 선택 시 문제 풀이 easy 40% + medium 40% + hard 20% 비율로 구성된다
- [ ] 4. `difficulty: "easy"` 선택 시 easy 문제만, `"hard"` 선택 시 hard 문제만 출제된다
- [ ] 5. 전체 모드에서 문제가 한국사 → 과학 → 지리 → 일반상식 라운드 로빈 순서로 배치된다
- [ ] 6. 세션 시작 시 문제 순서가 Fisher-Yates 알고리즘으로 셔플된다
- [ ] 7. 각 문제의 선택지 순서가 셔플되고 `answer` 인덱스가 새 위치에 맞게 갱신된다
- [ ] 8. `submitAnswer(selectedIndex)` 호출 시 정답 여부 채점, points 계산, `answers` 배열 기록이 이루어진다
- [ ] 9. 난이도별 점수가 easy +5점, medium +10점, hard +15점으로 정확히 계산된다
- [ ] 10. `nextQuestion()` 호출 시 마지막 문제면 `RESULT`, 아니면 `PLAYING` 상태로 전환된다

### UI 렌더링 (quiz.js)
- [ ] 11. 시작 화면에서 난이도 토글(초급/혼합/고급) 버튼이 동작하고 기본값이 혼합이다
- [ ] 12. 카테고리 버튼 5개(한국사/과학/지리/일반상식/전체) 클릭 시 해당 카테고리로 게임이 시작된다
- [ ] 13. 문제 화면에 카테고리 태그, 진행 번호(N/전체), 진행 바, 누적 점수가 표시된다
- [ ] 14. 진행 바의 너비가 `(currentIndex / questions.length) * 100%`로 정확히 계산된다
- [ ] 15. 선택지 클릭 후 모든 선택지 버튼이 즉시 `disabled` 처리되어 중복 클릭이 불가능하다
- [ ] 16. 피드백 오버레이에서 정답 시 헤더가 초록 배경 + "✔ 정답입니다! (+N점)"으로 표시된다
- [ ] 17. 피드백 오버레이에서 오답 시 헤더가 빨간 배경 + "✘ 오답입니다! (+0점)"으로 표시된다
- [ ] 18. 피드백 오버레이에서 정답 선택지는 초록, 오답으로 선택한 선택지는 빨간색으로 강조된다
- [ ] 19. 피드백 오버레이에 해당 문제의 해설 텍스트가 표시된다
- [ ] 20. `PLAYING`/`FEEDBACK` 상태에서 브라우저 새로고침 시 "게임을 종료하시겠습니까?" 확인 모달이 표시된다

## Step 3 — 결과/순위 화면 + UI 완성 + 반응형

### 결과 화면 (quiz.js)
- [ ] 1. 결과 화면에 최종 점수가 0에서 시작해 0.8초 카운트업 애니메이션으로 표시된다
- [ ] 2. 정답률(%)이 `(맞춘 문제 수 / 전체 문제 수) * 100`으로 정확히 계산되어 표시된다
- [ ] 3. 전체 모드 플레이 시 카테고리별 성취도 바 차트가 표시된다
- [ ] 4. 성취도 바가 CSS `transition width 0.6s ease 0.2s`로 채워지는 애니메이션이 동작한다
- [ ] 5. 닉네임 입력 시 정규식 `/^[a-zA-Z0-9가-힣]{1,10}$/` 유효성 검사가 실시간으로 동작한다
- [ ] 6. 유효하지 않은 닉네임 입력 시 저장 버튼이 비활성화되고 에러 메시지가 표시된다

### 순위 시스템 (leaderboard.js)
- [ ] 7. `saveScore()` 호출 시 점수가 `localStorage`의 `"quiz_leaderboard"` 키에 저장된다
- [ ] 8. 카테고리별 순위가 점수 내림차순으로 정렬되고 상위 10개만 유지된다
- [ ] 9. `localStorage` 용량 초과(QuotaExceededError) 발생 시 가장 오래된 항목 1개를 삭제 후 재저장한다
- [ ] 10. 용량 초과 처리 시 사용자에게 토스트 알림이 표시된다
- [ ] 11. 순위 화면 필터 탭(전체/한국사/과학/지리/일반상식) 클릭 시 해당 카테고리 순위로 재렌더링된다
- [ ] 12. 1~3위 행에 메달 아이콘(🥇🥈🥉)이 표시된다
- [ ] 13. 방금 저장된 순위 행이 연한 파란 배경으로 강조 표시된다

### CSS 완성 & 애니메이션
- [ ] 14. 피드백 오버레이 등장 시 `@keyframes fadeIn` 0.15초 애니메이션이 동작한다
- [ ] 15. 다음 문제 전환 시 `@keyframes slideLeft` 0.2초 애니메이션이 동작한다
- [ ] 16. 토스트 알림이 화면 하단 중앙에 나타났다가 3초 후 fade-out된다
- [ ] 17. 카테고리 버튼 hover 시 `translateY(-2px)` + box-shadow 증가 효과가 적용된다

### 반응형 & 통합
- [ ] 18. 모바일(480px 이하)과 데스크톱(768px 이상) 레이아웃이 각각 올바르게 표시된다
- [ ] 19. 게임 전체 플로우(시작 → 문제 → 피드백 → 결과 → 순위 → 홈)가 끊김 없이 동작한다
- [ ] 20. "다시 하기" 버튼으로 동일 카테고리/난이도로 재시작되고 새로운 문제 순서로 출제된다
