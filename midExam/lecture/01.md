## 바이브 코딩이란?
> "There's a new kind of coding I call **vibe coding**, where you fully give in to the vibes, embrace exponentials, and forget that the code even exists."
> — Andrej Karpathy (OpenAI 공동 창립자, 전 Tesla AI 리더) / 2025.2.2 X 포스팅

* **자연어(일상 언어)**로 "이런 걸 만들어 줘" 라고 말하면 AI가 코드를 생성해 주는 개발 방식
* 코드를 직접 타이핑하는 대신 → **대화와 피드백**으로 소프트웨어를 만든다
* 2025년 Collins 영어사전 올해의 단어로 선정될 만큼 전 세계적 화제

**핵심 포인트:** "AI가 대신 해 준다"가 아니라, **"AI와 대화하면서 함께 만든다"**

## 바이브 코딩의 기본 사이클
바이브 코딩은 한 번에 완벽하게 만드는 게 아니라 **반복 개선(Iteration)** 합니다.

[다이어그램: 아이디어 정의 → 프롬프트 작성 → 결과 확인 → 수정 요청 → 검토 및 기록의 순환 구조]

> **"왜 이 코드가 이렇게 동작하는지?"** - 스스로 설명할 수 있으면 진짜 실력!

## AI 통합 도구 - 모델 선택형
AI가 발전하면서 코딩을 도와주는 통합 도구가 등장했습니다. 바이브 코딩 도구는 크게 두 가지 유형으로 나뉩니다.

### 모델 선택형 도구
여러 AI 모델 중 원하는 것을 골라서 사용할 수 있는 도구

| 도구 | 특징 |
| :--- | :--- |
| **Cursor** | VS Code 기반 AI 코딩 에디터 |
| **Replit** | 웹 브라우저에서 바로 코딩 가능 |
| **Windsurf** | AI 기반 개발 환경 |

---

## AI 통합 도구 - 전용 모델형
### 전용 모델형 도구
자체 AI 모델을 가지고 있어 해당 모델만 사용하는 도구

| 도구 | 모델 |
| :--- | :--- |
| **Claude Code** | Claude (Anthropic) |
| **Gemini CLI** | Gemini (Google) |

* **정리:** 모델 선택형 = 여러 AI 선택 가능 / 전용 모델형 = 자체 AI만 사용
* 이 수업에서는 전용 모델형인 **Claude Code**를 주로 사용합니다.

---

## AI 어시스턴트의 발전 과정
`뉴럴 네트워크 (1950s~)` → `딥러닝 (2010s~)` → `트랜스포머 (2017~)` → `LLM (2020~현재)`

* **뉴럴 네트워크:** 인간의 뇌를 모방한 기초 인공지능
* **딥러닝:** 이미지·음성 인식 등 다양한 분야에서 성능 급상승
* **트랜스포머:** 문장 전체의 맥락을 한 번에 파악 → 언어 이해 능력 혁신
* **LLM (대규모 언어 모델):** 사람과 대화하듯 코드를 작성해 주는 AI

이 수업에서는 LLM 기반 AI 도구인 **클로드(Claude)**를 주로 사용합니다.

---

## 왜 클로드(Claude)인가?
### 코딩 분야 점유율 1위 - 41%
* 2025년 기준 AI 코딩 도구 시장에서 클로드가 41%로 가장 높은 점유율
* 2위 ChatGPT, 3위 Gemini를 크게 앞서는 수치

### 클로드의 강점
* **Artifacts:** 코드 실행 결과를 즉시 미리보기로 확인 가능
* **게시(Publish):** 만든 결과물을 링크로 바로 공유 가능
* **커스터마이즈(Customize):** 대화로 쉽게 디자인/기능 수정
* **무료 사용 가능:** 기본 기능 무료 (회원 가입만 하면 OK)

이번 학기 교재 **"혼자 공부하는 바이브 코딩 with 클로드 코드"**에서도 클로드를 기본 도구로 사용합니다.

## 바이브 코딩 시 주의할 점
바이브 코딩은 강력하지만, 꼭 기억해야 할 점이 있습니다.

* **코드를 이해하려고 노력하기:** AI가 만든 코드를 무조건 수용하지 말 것
* **보안에 주의하기:** AI 생성 코드에 보안 취약점이 있을 수 있음
* **검토는 필수:** 생성 → 검토 → 수정의 반복이 품질을 결정함
* **프로토타이핑에 효과적:** 빠른 시도와 실험에 최적화된 방법

> 이 수업에서는 **"AI가 만든 코드를 이해하고 설명할 수 있는 능력"**이 평가의 핵심입니다.

## 실습 ③ 게시(Publish) 하기
완성된 웹페이지를 다른 사람에게 공유해 봅시다[cite: 124].

1. 아티팩트 미리보기 상단의 **게시(Publish)** 버튼 클릭 [cite: 126]
2. 팝업에서 **게시 및 링크 복사** 클릭 [cite: 127]
3. 복사된 링크를 브라우저 새 탭에 붙여 넣기 [cite: 128]

새 탭에서 내가 만든 웹페이지가 정상적으로 보이면 성공입니다! [cite: 129]
