## 단계별 프롬프트 전략이란? [cite: 849-851]
PRD를 기반으로 프로젝트를 여러 단계로 나누어 AI에게 구체적인 목표를 제시하는 방식입니다.

### 기획과 구현의 분리 [cite: 852]
* **기획 (클로드 웹):** PRD 작성
* **구현 (클로드 코드):** 뼈대 → 기능 → UI → 고급 → 최적화

### 왜 단계별로 나눠야 할까? [cite: 853-857]
* 한 번에 모든 기능을 요청하면 품질이 떨어짐
* 각 단계의 결과를 확인한 뒤 다음 단계 진행 가능
* 이전 단계의 결과물 위에 점진적으로 기능 추가

## 세션(Session)이란? [cite: 957-963]
클로드 코드에서 대화를 시작해 명령을 주고받는 동안 유지되는 상태를 말합니다. 터미널을 닫아도 대화 기록은 프로젝트 폴더에 자동 저장되므로 **세션 재개**가 필수적입니다. [cite: 958-962]

## 세션 재개하기 (교재 p.148~149)
장기 프로젝트 진행 시 이전 작업 상태를 이어서 하는 방법입니다[cite: 965, 963].

### 방법 1: `claude --resume`
* 이전 대화 기록을 불러와서 마지막 상태 그대로 이어서 작업할 수 있습니다 [cite: 967-968].

### 방법 2: `/resume` 명령어 (별칭: `/continue`)
* `claude` 실행 후 터미널에 `/resume`을 입력하면 이전 대화 목록 중 하나를 선택하여 재개할 수 있습니다 [cite: 969-970].

### 방법 3: 최근 대화 즉시 복구 (`--continue` 또는 `-c`)
* `claude --continue` 명령어를 사용하면 가장 최근의 마지막 대화를 자동으로 복구합니다 [cite: 973-975].

[이미지: 클로드 코드에서 세션 재개 목록을 보여주는 터미널 화면] [cite: 976-1001]

## 자동 승인 모드 (교재 p.150)
반복적인 파일 수정 승인 절차를 간소화하는 기능입니다[cite: 1002, 1006].

* **전환 방법:** `Shift + Tab` 키를 눌러 모드를 전환합니다[cite: 1003, 1007].
* **특징:** 파일 수정 승인을 매번 누르지 않아도 AI가 자동으로 처리하여 작업 속도를 향상시킵니다 [cite: 1005-1006].
* **주의:** 편리하지만 코드 변경 사항을 나중에 반드시 점검해야 합니다[cite: 1010].

## 컨텍스트(Context)란? (교재 p.153~154)
현재 대화와 작업 정보를 저장 및 표시하는 메모리 공간입니다 [cite: 1012-1013]. 대화가 길어져 컨텍스트가 부족해지면 '자동 압축(auto-compact)' 경고가 발생할 수 있습니다[cite: 1014].

### `/context`: 컨텍스트 상태 확인
* 터미널에 `/context`를 입력하여 현재 사용량을 시각적으로 확인할 수 있습니다 [cite: 1015-1016].
* 사용량은 시스템 프롬프트, 도구, 메모리 파일, 메시지 등으로 구분되어 표시됩니다 [cite: 1020-1025].

[이미지: 터미널에서 `/context` 명령어를 입력했을 때 나타나는 컬러 그리드와 토큰 사용량 정보] [cite: 1012, 1019]

### `/compact`: 컨텍스트 압축하기 (교재 p.155~156)
* 대화 기록을 요약하여 컨텍스트 공간을 확보합니다 [cite: 1033-1035].
* 압축 후에도 `CLAUDE.md`와 프로젝트 파일 정보는 유지됩니다[cite: 1036].

## `@` 기호로 파일 및 폴더 분석하기 (교재 p.164~166)

* **폴더 분석:** `@web_version 폴더에 있는 파일들을 분석해서 데스크톱 환경 최적화 여부를 확인해 줘.` [cite: 1074]
* **장점:** 파일 전체를 훑지 않아도 필요한 정보를 즉시 참조할 수 있어 효율적입니다[cite: 1070].

## 핵심 키워드 정리 [cite: 1197]
### 이론 키워드 [cite: 1198-1205]
* **단계별 프롬프트 전략:** PRD를 기반으로 프로젝트를 여러 단계로 나누어 목표를 제시하는 방식
* **PRD:** 제품의 목적, 기능, 기술 스펙을 체계적으로 정리한 문서
* **세션(Session):** 대화가 시작되어 종료될 때까지 유지되는 상태
* **컨텍스트(Context):** 현재 대화와 작업 정보를 저장하는 메모리 공간
* **멀티모달 AI:** 텍스트뿐만 아니라 이미지 등을 함께 인식하는 AI

### 실습 키워드 [cite: 1206-1217]
* **--resume:** 이전 대화 목록에서 세션을 선택해 복원
* **--continue (-c):** 가장 최근 대화를 자동으로 복구
* **자동 승인 모드:** `Shift + Tab`으로 권한 승인 절차를 생략
* **@ 기호:** 파일 및 폴더를 즉시 참조하여 분석
* **/doctor:** 시스템 상태(버전, 설치 방식 등) 진단
* **/export:** 작업 기록을 문서화하여 저장
