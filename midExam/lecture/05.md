## 할루시네이션이란? (교재 p.174) [cite: 1295-1299]
**할루시네이션(hallucination, 환각)**은 생성형 AI가 사실과 다른 내용을 진짜처럼 만들어내는 현상입니다.

* **발생 원인:** AI가 학습 데이터에서 패턴을 찾아 확률적으로 그럴듯한 문장을 생성하거나, 불명확한 정보로 인해 추론 오류를 범하기 때문입니다.
* **대처 방법:** [cite: 1300-1304]
    * 신뢰할 수 있는 문서 또는 공식 사이트와 비교해 **교차 검증**을 수행합니다.
    * AI에게 "출처 없는 정보는 생성하지 말 것"과 같은 규칙을 입력하여 사전 방지합니다.

> **주의:** 사실 기반 콘텐츠(퀴즈, 역사, 과학)는 반드시 교차 검증이 필요합니다! [cite: 1303]

## 할루시네이션 해결: 교차 검증 가이드라인 [cite: 1343-1351]
애매한 표현("가장 긴", "최초의")을 발견하고 명확한 기준(전통적 측정 기준 등)을 추가하는 것이 **추상화 사고력**입니다.

### 가이드라인 구축 (`CLAUDE.md`) [cite: 1353-1375]
`/memory` 명령어를 사용하여 `Project memory (./CLAUDE.md)`에 아래 내용을 추가합니다.

1. **정답의 유일성:** 다른 해석 가능 시 조건 명시 (예: 면적 기준) [cite: 1365-1366]
2. **최상급 표현의 기준:** '가장 큰' 등에는 측정 기준 명시 [cite: 1367-1368]
3. **명확한 시간/범위:** 변할 수 있는 정보는 시점(예: 2024년 기준) 명시 [cite: 1369-1371]
4. **교차 검증:** 의심스러운 정보는 2개 이상 출처 확인 및 주류 학설 기준 적용 [cite: 1372-1374]

## 커스텀 명령어란? (교재 p.191, p.194) [cite: 1416-1417]
**커스텀 명령어(custom command)**란 사용자가 직접 정의해서 쓸 수 있는 나만의 단축 지시어입니다.

### 왜 필요한가? [cite: 1418-1420]
* 문제 형식 통일, 가이드라인 검증, 난이도 균형 확인 등 반복 작업의 효율을 높이기 위함입니다.
* 매번 긴 프롬프트를 입력할 필요가 없어 생산성이 향상됩니다.

### 커스텀 명령어의 종류 [cite: 1421-1422]
| 종류 | 저장 위치 | 범위 |
| :--- | :--- | :--- |
| **전역 명령어** | `~/.claude/commands/` | 모든 프로젝트에서 사용 가능 |
| **프로젝트 명령어** | `.claude/commands/` | 해당 프로젝트에서만 사용 가능 |

## 커스텀 명령어란? (교재 p.191, p.194) [cite: 1416-1417]
**커스텀 명령어(custom command)**란 사용자가 직접 정의해서 쓸 수 있는 나만의 단축 지시어입니다.

### 왜 필요한가? [cite: 1418-1420]
* 문제 형식 통일, 가이드라인 검증, 난이도 균형 확인 등 반복 작업의 효율을 높이기 위함입니다.
* 매번 긴 프롬프트를 입력할 필요가 없어 생산성이 향상됩니다.

### 커스텀 명령어의 종류 [cite: 1421-1422]
| 종류 | 저장 위치 | 범위 |
| :--- | :--- | :--- |
| **전역 명령어** | `~/.claude/commands/` | 모든 프로젝트에서 사용 가능 |
| **프로젝트 명령어** | `.claude/commands/` | 해당 프로젝트에서만 사용 가능 |
